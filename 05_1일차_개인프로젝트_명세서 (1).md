# [개인 프로젝트]: "I AM"

> 본 프로젝트는 바이브 코딩 혹은 본인의 개발 역량을 활용하여, 주어진 시간 안에 하나의 완성된 웹사이트를 만
> 드는 개인 프로젝트입니다.

> Gemini Canvas 안내
> Gemini Canvas 의 경우 사용량이 무한하지 않습니다.
> 사용 중 Gemini Canvas 할당량이 다 된 경우 Canvas 가 아닌 일반 채팅으로 전환하여 코드 생성 요청을 합니다.
> 그리고 VS Code 에서 새롭게 html 파일을 생성한 뒤 생성된 코드를 붙여넣어 주는 방식으로 이어서 하면 지속적
> 인 개발이 가능합니다.

## 1. 프로젝트 개요

- **프로젝트명:** I AM (아이엠)
- **한 줄 정의:** 내가 좋아하는 주제 (나 자신, 포트폴리오, 취미, 반려동물, 등) 를 매력적으로 소개하는 웹 서비스
- **목적:** 자신이 보여주고 싶은 내용을 AI 를 활용하여 화면을 구현하고, 그 과정에서 개발의 재미와 성취를 느끼기
- **주제 선택 (예시):**
  
  - **포트폴리오**: 나를 소개하는 이력서 및 개인 프로필 페이지
  - **취미**: 내가 좋아하는 영화, 음악, 운동, 게임 등을 정리한 페이지
  - **데이터 큐레이션**: 나만 알고 있는 로컬 맛집 리스트, 유용한 개발 툴 추천 가이드
  - **자랑 페이지**: 우리 집 반려동물 소개, 내가 만든 창작물 전시 페이지
  - 위 예시 외에도 자신이 소개하고 싶은 흥미로운 모든 컨텐츠
- **활용 도구**
  
  - Gemini Canvas 를 활용 도구로 합니다.
  - 할당량이 만료된 경우 Gemini & VS Code 를 도구로 활용합니다.
  
  

## 2. 진행 구성

이 프로젝트는 **필수 요구사항** 와 **심화 요구사항**으로 이틀간 기능을 구현해야합니다. 모든 교육생은 공통 파트로 필수 요
구사항을 먼저 완성한 뒤, 이어서 심화 요구사항에서 서술된 요구사항을 구현합니다.



## 3. 개발 요구사항

아래 우선순위에 따라 개발을 진행합니다.



### 필수 요구사항 (1일차)

- **단일 페이지 구성**: 선택한 주제의 핵심 내용이 한눈에 들어오는 메인 화면

- **콘텐츠**: 주제를 잘 전달할 수 있는 텍스트, 이미지 등과 함께 기본 상호작용 (버튼 클릭 리액션, 팝업 창 등) 구현

- **바닐라 JS 활용**: HTML + CSS + JS 구조로 구현 (별도 프레임워크 · 빌드 도구 없이)

  

### 심화 요구사항 (2일차)

- 공통 요구사항
  - **모바일 친화 기능**: PC 모니터뿐만 아니라 모바일 화면에서도 디자인이 깨지지 않고 자연스럽게 보이도록 구현
  - **Git 으로 코드 관리**: 버전별로 코드를 기록하며 관리

  

- **추가 요구사항**
  
  - 1 일차에 만든 단일 페이지(SPA) 결과물을 상단 네비게이션 바로 서로 연결되는 MPA(Multi Page Application) 구조로 재구성
    - Canvas 에서 페이지를 하나씩 만들기
    - 확정된 네비게이션 바 코드를 모든 페이지에 동일하게 삽입
    - VS Code 로 가져와 파일로 조립하고 로컬에서 링크 이동 테스트
  - **게이미피케이션 요소**: 포인트, 진행률, 퀴즈, 뱃지 등 참여형 요소 추가
  - 위 요구사항 외에 자신이 구현하고 싶은 모든 기능
  
  
  
  

###  용어 설명

- **SPA (Single Page Application)**: 페이지 이동 없이 한 화면 안에서 콘텐츠가 바뀌는 웹사이트 구조

- **MPA (Multi Page Application)**: 여러 개의 HTML 파일이 링크로 연결된 전통적인 웹사이트 구조

- **바닐라 JS**: React, Vue 같은 프레임워크 없이 순수 자바스크립트만 사용하는 것

- **게이미피케이션**: 포인트, 뱃지, 진행률처럼 게임 요소를 넣어 참여를 유도하는 기법
