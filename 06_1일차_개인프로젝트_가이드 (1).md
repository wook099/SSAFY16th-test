# I AM — Gemini Canvas로 내가 고른 주제를 소개하는 나만의 공간

> **프로젝트명:** I AM (아이엠)
>
> **한 줄 정의:** 내가 좋아하는 주제 (나 자신, 포트폴리오, 취미, 반려동물 등) 를 매력적으로 소개하는 단일 페이지 웹 서비스
>
> **목적:** 보여주고 싶은 내용을 AI 의 도움으로 화면으로 구현하며, 개발의 재미와 성취감을 느끼기
>
> **트랙:** HTML5 + CSS + Vanilla JS 단일 파일로, Gemini Canvas 에 화면 구성을 요청하고 디자인을 직접 수정해보기
>
> **목표:** 시간 안에 바닐라 JS 기반 1 페이지 (SPA) 소개 사이트 기본형 완성
>
> **사용 기술:** Vanilla JS (순수 JS) / CSS / HTML
>
> **활용 도구:** 
>
> * Gemini Canvas (기본 활용 도구)
> * Gemini & VS Code (할당량이 만료된 경우)

------



## 주제 선택 (예시)

아래 예시 외에도 자신이 소개하고 싶은 흥미로운 모든 콘텐츠가 주제가 될 수 있습니다.
- **포트폴리오:** 나를 소개하는 이력서 및 개인 프로필 페이지
- **취미:** 내가 좋아하는 영화, 음악, 운동, 게임 등을 정리한 페이지
- **데이터 큐레이션:** 나만 알고 있는 로컬 맛집 리스트, 유용한 개발 툴 추천 가이드
- **자랑 페이지:** 우리 집 반려동물 소개, 내가 만든 창작물 전시 페이지

## 필수 요구사항 구현

- **단일 페이지 구성:** 선택한 주제의 핵심 내용이 한눈에 들어오는 메인 화면
- **콘텐츠:** 주제를 잘 전달하는 텍스트 + 기본 상호작용 (버튼 클릭 리액션, 팝업 등)
> 참고 - 사용전 안내
>
> Gemini Canvas 는 프롬프트 한 번으로 실행 가능한 코드 (앱) 를 바로 만들어주는 도구입니다.
>
> 다만 " 알아서 예쁘게 만들어줘 " 식의 막연한 프롬프트는 결과물이 부실합니다.
>
> Canvas 가 잘 반응하는 프롬프트는 보통
>
> 페르소나 (누구에게 요청하는가) → 작업 (무엇을 만드는가) →
> 맥락 (왜, 어떤 재료로) → 형식 (어떤 산출물 형태로)
>
> 4 단 구조를 따를 때 완성도가 올라갑니다.
>
> 아래 템플릿도 이 구조를 그대로 사용합니다.
>
> 시간을 고려해 1 차 프롬프트 (뼈대) → 2~3 차 수정 프롬프트 (디테일) 로 나눠서 넣는 걸 추천합니다.
> 처음부터 모든 걸 한 번에 요청하면 Canvas 가 절반만 반영하고 나머지는 흘려버리는 경우가 많습니
> 다.
>
> 이 문서에서 다루는 범위는 기본형까지입니다. 
>
> 게이미피케이션, CSS 애니메이션 도전형, 정적 사진 업그레이드 같은 심화 기능은 이 문서에서는 다루지 않으니, 먼저 기본형을 확실히 완성하는 데 집중하세요.

## 1.기획 프롬프트

* 웹사이트에 들어갈 문구와 톤을 AI에게 맡기기 전에, 무엇을 어떤 순서로 보여줄지 사람이 먼저 정리하는 과정입니다. 
* 코드는 전혀 만들지 않고, 대화를 통해 캐치프레이즈, 매력 포인트, 타겟, 톤을 표 하나로 뽑아내는 게 목표입니다.

```
[페르소나]
너는 브랜딩 기획자야. 웹사이트 제작 전에 컨셉을 정리하도록 도와줘. 코드는 아직 쓰지 마.

[정보]
- 소개 대상: {{ 나 자신 / 포트폴리오 / 취미 / 반려동물 / 큐레이션 등 }}
- 자유롭게 적은 내용: {{ 편하게 나열 }}

[요청]
아래 6가지를 하나씩 질문하고, 답변이 끝나면 표로 정리해줘.
1) 캐치프레이즈 후보 3개
2) 매력 포인트 3~5개 (구체적으로)
3) 타겟 방문자
4) 스토리텔링 흐름
5) 무드/톤 (형용사 3개)
6) 메인 컬러 + 폰트
```

**주의할 점**

- 매력 포인트는 "좋다", "잘한다" 같은 막연한 표현 대신 숫자나 에피소드로 채우기 
  (막연한 답을 주면 2단계 결과물도 막연해집니다.)
- 타겟 방문자를 너무 넓게 잡지 않기 
  (예: "모든 사람"이 아니라 "함께 일할 팀원" 처럼 좁혀서)
- 대화가 길어지지 않게 주의하기
  (대화가 길어지면 내용을 빠뜨릴 수 있음, 만약 대화가 길어지면 새로운 채팅을 시작하는 것이 좋음)
- 결과로 나온 기획서 표는 반드시 메모장 등에 따로 저장하기 
  (2단계 진행 중 잊어버리기 쉬움)

## 2. 코드 생성 프롬프트

* 이전 단계에서 만든 기획서를 그대로 프롬프트에 붙여넣어 실제 HTML/CSS/JS 코드를 받는 과정입니다. 
* 여기서는 AI가 "무엇을 만들지"가 아니라 "어떻게 구현할지"만 고민하도록, 결정은 이미 이전 단계에서 끝낸 상태로 넘어갑니다.

```
[페르소나]
너는 바닐라 HTML/CSS/JS 프론트엔드 개발자야. 프레임워크·빌드도구 없이 작업해줘.

[기획서]
{{ 1단계 결과 그대로 붙여넣기 }}

[요청]
기획서 기반으로 1페이지 웹사이트를 만들어줘.
구조: 히어로 → 소개 → 강점 카드 → (선택) CTA → 푸터
톤과 문구는 기획서의 "스토리텔링 흐름"과 "무드/톤"을 따라가줘.

[형식]
Canvas에서 바로 미리보기 가능하게. 전체 구조 먼저 만들고, 이후 디테일 다듬기.
```

**주의할 점**

- 세부 수정은 한 번에 하나씩, 범위를 좁혀서 요청하기 
  - "버튼 색만 바꿔줘"처럼 구체적으로. 
  - 여러 가지를 한꺼번에 바꿔달라고 하면 멀쩡했던 다른 부분까지 같이 깨지는 경우(사이드 이펙트)가 흔합니다
- 수정 요청 후에는 이전에 잘 됐던 부분이 그대로인지 전체를 다시 훑어보기
- 이미지/폰트는 출처가 분명한 무료 리소스만 사용하기 (저작권 문제 예방)

## 3. 예시: "SSAFY "소개 페이지 만들기

```
너는 바닐라 HTML/CSS/JavaScript 로 반응형 원페이지 웹사이트를 만드는 프론트엔드 개발자야. 
프레임워크나 외부 빌드 도구 없이 순수 코드로만 작업해줘.

아래 대상을 소개하는 1 페이지 웹사이트를 만들어줘.
- 대상: SSAFY (청년 대상 소프트웨어 ·AI 교육 프로그램)
- 캐치프레이즈: " 배움에서 성장으로, 성장에서 준비로 이어지는 SSAFY"
- 핵심 매력 포인트 3 가지:
  1. 배움 - 기초 개념부터 차근차근 익혀서 처음 시작해도 흐름을 놓치지 않음
  2. 성장 - 프로젝트와 협업을 통해 배운 내용을 실제 상황에 적용
  3. 준비 - 교육이 끝난 뒤에도 포트폴리오 · 취업 준비로 자연스럽게 이어짐
- 타겟 방문자: SSAFY 지원을 고민하는 예비 지원자, 청년 구직자

페이지는 아래 순서로 구성해줘.
  1. 히어로 - "SSAFY" 타이틀 + 캐치프레이즈 + 첫인상용 그래픽 영역
  2. 소개 - "SSAFY 는 어떤 프로그램인가요 ?" 청년들의 소프트웨어 ·AI 역량 향상과 취업 준비를 함께 돕는 교육 프로그램이라는 점을 설명
  3. 매력 포인트 카드 3 개 (배움 / 성장 / 준비)
  4. 교육 과정 흐름 섹션 - " 기초 → 적용 → 프로젝트 " 로 이어지는 학습 흐름을 3 단계 카드나 타임라인 형태로 보여주기
  5. 지원 포인트 섹션 - 학습 / 적용 / 협업 / 진로, 4 가지를 카드로 배치
  6. 갤러리 - 프로그램 관련 이미지 6 장 그리드 (우선 회색 placeholder 로)
  7. CTA - " 지원 안내 더 알아보기 " 버튼. 클릭하면 살짝 확대되는 반응 효과를 주고 5 번 섹션으로 부드럽게 스크롤 이동시켜줘.
  8. 푸터 - 공식 사이트 링크 / 문의처

디자인 톤은 신뢰감 있고 깔끔한 느낌 (딥블루 + 화이트 위주), 제목은 굵고 임팩트 있는 폰트로 작성해줘.
```



## 4. 체크리스트

> 체크 리스트를 만족하지 않았다면, 다시 Gemini 에 요청하여 적용할 수 있도록 합니다.

- [ ] 단일 페이지 안에 주제의 핵심 내용이 한눈에 들어오는 구성인가 (필수 요구사항)

- [ ] 버튼 클릭 리액션이나 팝업 같은 기본 상호작용을 최소 1 개 넣었는가 (필수 요구사항)

- [ ] 템플릿 빈칸을 채우고 1 차 프롬프트로 뼈대를 생성했는가

- [ ] 섹션 순서 / 문구 / 컬러 등 내용을 다듬었는가

- [ ] 기본 CSS 애니메이션 (페이드인 등) 을 적용하고 placeholder 를 정적 사진으로 교체했는가

- [ ] 페르소나 / 작업 / 맥락 / 형식 4 단 구조로 1 차 프롬프트를 작성했는가
- [ ] 홍보 대상의 캐치프레이즈와 매력 포인트를 구체적으로 채워 넣었는가
- [ ] Canvas 미리보기의 콘솔에 에러가 없는지 확인했는가
- [ ] 1 차 뼈대 (HTML 구조) 가 생성된 후 다운로드 받아 직접 브라우져에서 확인했는가



## 5. 용어 설명

- **SPA (Single Page Application):** 페이지 이동 없이 한 화면 안에서 콘텐츠가 바뀌는 웹사이트 방식. 여기서는 " 스크롤만으로 다 보이는 한 페이지짜리 사이트 " 라는 뜻으로 사용했습니다.

- **바닐라 JS:** React, Vue 같은 별도 프레임워크 없이 순수 자바스크립트만 사용하는 방식.

- 게이미피케이션 (Gamification): 게임이 아닌 서비스에 포인트, 진행률, 배지처럼 게임적 요소를 넣어 재미와 몰입감을 주는 기법.

- **CSS 애니메이션:** 버튼이 눌리거나 요소가 스르륵 나타나는 등, 코드 없이 CSS 속성만으로 만드는 화면 움직임 효과.

- **placeholder(플레이스홀더) 이미지:** 실제 이미지가 준비되기 전 자리만 잡아두는 임시 이미지 (보통 회색 박스).

- **스켈레톤 UI:** 콘텐츠가 로딩되는 동안 실제 내용 대신 회색 뼈대 모양을 먼저 보여주는 로딩 표현 방식.
